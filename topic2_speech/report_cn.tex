\begin{abstract}
本文重构 VE216 Project Topic 2 的主线，不再把重点放在神经网络本身，而是放在任务定义、假设、理论分析、算法结构和定性案例上。我们把原先较松的词级多标签实验，收紧为一个更接近“首浊辅音检测”的四类任务：只保留首字母为 \texttt{g/b/d/z} 且 CMUdict 中首个非元音音素严格匹配 \texttt{G/B/D/Z} 的单词，共得到 60 个词、3000 条 16 kHz 语音。方法上，本文保留课内模板法作为主方法，并用 MLP 和 CNN 作为对照基线。结果表明，课程模板法可解释但误报高；MLP 的 exact-match 最好；CNN 的 macro-F1 最好。更重要的是，四类辅音的差异和声学事实一致：/z/ 最容易，/g/ 最难。
\end{abstract}

\section{问题定义与报告主线}

\subsection{任务形式化}

设输入语音为 $x_i$，目标标签为 $y_i \in \{0,1\}^4$，对应标签集合
\[
C=\{g,b,d,z\}.
\]
在当前数据集上，每个样本恰好属于四类中的一类，因此满足 $\sum_{c\in C} y_{i,c}=1$。但实现里仍保留一对多接口：先对每个标签输出分数 $s_{i,c}$，再用该标签自己的阈值 $\tau_c$ 做判定
\[
\hat y_{i,c} = \mathbb{1}[s_{i,c}\ge \tau_c].
\]
这样做的原因是：课内模板法、统计特征分类器和 CNN 输出的都不是天然校准过的概率，统一用 dev 集选阈值更公平，也更容易比较不同方法的特性。

\subsection{本文的重点}

这份报告只把 AI 模型当作对照基线，不把它们当成主角。主线是：
\begin{itemize}[nosep]
    \item 任务边界是否真的收紧到了“首浊辅音检测”；
    \item 课内模板法到底在假设什么、怎么计算、为什么会误报；
    \item 统计特征和时频特征为什么能比模板法更稳；
    \item /g/、/b/、/d/、/z/ 为什么会呈现出明显不同的难度。
\end{itemize}

\section{数据集重建与标注规则}

\subsection{筛选规则}

新数据集由 \path{src/prepare_mswc_initial_gbdz_subset.py} 生成，核心规则只有一条：
\begin{quote}
保留一个单词，当且仅当它的首字母属于 \texttt{g/b/d/z}，并且 CMUdict / ARPABET 中首个非元音音素与该字母严格对应为 \texttt{G/B/D/Z}。
\end{quote}

这条规则的意义是把“字母里出现了某个浊辅音”改成“这个浊辅音确实出现在起始位置”。这样可以自动去掉两类噪声样本：
\begin{itemize}[nosep]
    \item 拼写里有目标字母，但发音位置不对；
    \item 首字母对了，但真正起始音素不是目标浊辅音。
\end{itemize}

\begin{verbatim}
for each word w in MSWC:
    if first letter of w not in {g,b,d,z}: discard
    phones = CMUdict(w)
    p = first non-vowel phone in phones
    if p == first letter(w).upper():
        keep w
    else:
        discard
\end{verbatim}

\section{理论背景与方法}

\subsection{短时表示、局部窗与对比模板}

对 1 秒左右的短语音，真正有区分力的不是整词平均谱，而是词首浊辅音附近的局部时频结构。本文的前端仍然是标准短时分析：25 ms 帧长、10 ms 帧移、512 点 FFT、40 维 log-mel。不同的是，我们不再直接拿整词均值当模板，而是先把每帧 log-mel 压成能量曲线，平滑后用首次超过“基线 + margin”的位置当作 onset，再围绕 onset 截取固定长度窗口。

这个改造依赖两个假设：
\begin{itemize}[nosep]
    \item 目标辅音的判别证据主要集中在 onset 前后很短的一段时间里；
    \item 单独使用正模板不够，必须把“像目标”与“像干扰”同时建模。
\end{itemize}

因此，课内方法把每个样本变成两类局部描述：一类是窗口内分段均值拼接得到的 patch 向量，另一类是局部能量包络的自相关。训练时对每个标签分别学习正模板 $T_c^+$、负模板 $T_c^-$，以及正自相关模板 $A_c^+$、负自相关模板 $A_c^-$. 第 $i$ 个样本在平移 $k$ 下的得分定义为
\[
s_{i,c}^{(k)}=
0.65\bigl(\cos(p_i^{(k)},T_c^+)-\cos(p_i^{(k)},T_c^-)\bigr)
+
0.35\bigl(\cos(a_i^{(k)},A_c^+)-\cos(a_i^{(k)},A_c^-)\bigr).
\]
最终分数取候选平移上的最大值，$K=\{-2,0,2\}$。如果数据是单标签任务，就直接取 $\arg\max_c s_{i,c}$；如果保留一对多接口，则仍然可以回退到每类自己的阈值 $\tau_c$。

\subsection{算法 1：课程模板法（局部窗对比模板）}

\begin{verbatim}
Algorithm 1 局部窗对比模板法
Input: train/dev/test waveforms, labels C={g,b,d,z}
Output: per-label predictions on test

for each class c in C do
    collect positive and negative training samples
    build local patch templates T_c^+, T_c^- from onset-centered windows
    build local autocorr templates A_c^+, A_c^- from window envelopes
end for

for each sample i do
    detect onset from smoothed frame energy
    extract candidate windows around onset with shifts K={-2,0,2}
    for each class c in C do
        score each candidate by contrastive template similarity
        take the best candidate score as s_ic
    end for
    if single-label task then
        predict argmax_c s_ic
    else
        choose tau_c on dev and threshold
    end if
end for
\end{verbatim}

这个方法的核心不是“学一个复杂边界”，而是把判别问题改写为“局部窗里到底像不像目标辅音”。正样本模板提供典型形态，负样本模板提供干扰背景，两者之差比单独用正模板更稳。

\subsection{算法 2：统计特征 MLP 基线}

MLP 不是主角，只是一个强一点的统计基线。它的输入不是原始波形，而是手工汇总后的 62 维统计特征：
\begin{itemize}[nosep]
    \item 13 维 MFCC 的均值和标准差；
    \item MFCC 一阶差分的均值和标准差；
    \item 10 个 coarse band-energy 特征。
\end{itemize}

模型结构是 \texttt{StandardScaler -> MLPClassifier(128, 64)}，用 early stopping 训练。和模板法相比，它不再依赖显式模板对齐，而是直接学习这些统计特征之间的组合关系。因为当前数据是单标签四分类，但阈值仍按 label 单独选，所以 MLP 的 exact-match 往往会比 label-wise accuracy 更能反映它是否“少乱报”。

\subsection{算法 3：CNN 对照基线}

CNN 只作为更强的对照上界。输入是整词 log-mel spectrogram，网络是 3 个卷积块加自适应池化，再接一层小型全连接头。训练时使用带 \texttt{pos\_weight} 的 \texttt{BCEWithLogitsLoss}，并按 dev macro-F1 早停回退到最佳 epoch。

最终采用的调参配置是：
\[
\texttt{epochs}=10,\quad
\texttt{batch}=192,\quad
\texttt{lr}=10^{-3},\quad
\texttt{weight\_decay}=10^{-4},\quad
\texttt{patience}=5.
\]
它的作用只是提供一个“模型能学到多少局部时频模式”的参照，不影响本文的主要论证。

\section{讨论与局限}

这次重构最重要的收益，不是某个分数上升了，而是任务边界终于变清楚了。旧版更像“词里有没有出现某个浊辅音”，新版则更接近“词首这个浊辅音能不能被检出”。一旦把这个条件收紧，报告里的理论解释就顺了：数据集、特征、算法和案例分析都围绕同一个 onset 问题展开。

这套方法仍然有三个局限：
\begin{itemize}[nosep]
    \item 语音本身仍是整词级音频，不是人工切好的 phone segment；局部窗只是用 onset 近似对齐。
    \item /z/ 的词数仍然是数据集构造的瓶颈，想继续扩规模就得放宽单词池或换更大的语料。
    \item 目前的局部窗仍依赖阈值式 onset 检测，/g/ 和 /d/ 这类短爆破在对齐偏一点时就会掉分。
\end{itemize}

\section{结论}

本文完成了 Topic 2 的一次重新组织，核心结论如下：
\begin{itemize}[nosep]
    \item 通过“首字母与首个非元音音素严格对应”的规则，新数据集把任务收紧到了首浊辅音检测；
    \item 局部窗对比模板法在当前四类任务上取得最佳整体结果，macro-F1 为 0.546，exact-match 为 0.550；
    \item MLP 的 macro-F1 为 0.526，CNN 为 0.222，说明更强的深度模型并没有自动取代局部窗方法；
    \item /z/ 最容易、/g/ 最难，这和声学图里的持续摩擦与短促爆破完全一致；
    \item 这份报告的重点已经从“跑一个神经网络”转成“解释为什么这些辅音会这样表现”。
\end{itemize}
