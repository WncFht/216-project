\begin{abstract}
本文重构 VE216 Project Topic 2 的主线，不再把重点放在神经网络本身，而是放在任务定义、频谱描述、检测思路和定性案例上。我们把原先较松的词级多标签实验，收紧为一个更接近“首浊辅音检测”的四类任务：只保留首字母为 \texttt{g/b/d/z} 且 CMUdict 中首个非元音音素严格匹配 \texttt{G/B/D/Z} 的单词，共得到 60 个词、3000 条 16 kHz 语音。方法上，本文明确分成两层：第一层是严格课内版本，只使用 lecture slides 范围内的短时截取、DFT/FFT、幅度谱和相关检测，并给出可运行的 MATLAB 代码；第二层是保留现有 \texttt{local\_patch\_v1}、MLP 和 CNN 结果作为课外扩展与 AI 对照。这样，报告的主线重新回到“如何用课堂工具描述 /g/、/b/、/d/、/z/ 的频谱，并据此完成检测”。
\end{abstract}

\section{问题定义与报告主线}

\subsection{任务形式化}

设输入语音为 $x_i$，目标标签为 $y_i \in \{0,1\}^4$，对应标签集合
\[
C=\{g,b,d,z\}.
\]
在当前数据集上，每个样本恰好属于四类中的一类，因此满足 $\sum_{c\in C} y_{i,c}=1$。但实现里仍保留一对多接口：先对每个标签输出分数 $s_{i,c}$，再用该标签自己的阈值 $\tau_c$ 做判定
\[
\hat y_{i,c} = \mathbb{1}[s_{i,c}\ge \tau_c].
\]
这样做的原因是：严格课内模板法、课外扩展版模板法、统计特征分类器和 CNN 输出的都不是天然校准过的概率，统一用 dev 集选阈值更公平，也更容易比较不同方法的特性。

\subsection{本文的重点}

这份报告只把 AI 模型当作对照基线，不把它们当成主角。主线是：
\begin{itemize}[nosep]
    \item 任务边界是否真的收紧到了“首浊辅音检测”；
    \item 严格课内的 FFT 频谱模板法到底在假设什么、怎么计算、为什么能完成题目要求；
    \item 课外扩展版为什么会比严格课内版更稳，以及它究竟超出了哪些课堂边界；
    \item /g/、/b/、/d/、/z/ 为什么会呈现出明显不同的难度。
\end{itemize}

\section{数据集重建与标注规则}

\subsection{筛选规则}

新数据集由 \path{src/prepare_mswc_initial_gbdz_subset.py} 生成，核心规则只有一条：
\begin{quote}
保留一个单词，当且仅当它的首字母属于 \texttt{g/b/d/z}，并且 CMUdict / ARPABET 中首个非元音音素与该字母严格对应为 \texttt{G/B/D/Z}。
\end{quote}

这条规则的意义是把“字母里出现了某个浊辅音”改成“这个浊辅音确实出现在起始位置”。这样可以自动去掉两类噪声样本：
\begin{itemize}[nosep]
    \item 拼写里有目标字母，但发音位置不对；
    \item 首字母对了，但真正起始音素不是目标浊辅音。
\end{itemize}

\begin{verbatim}
for each word w in MSWC:
    if first letter of w not in {g,b,d,z}: discard
    phones = CMUdict(w)
    p = first non-vowel phone in phones
    if p == first letter(w).upper():
        keep w
    else:
        discard
\end{verbatim}

\section{理论背景与方法}

\subsection{严格课内方法：短时幅度谱模板}

如果严格限制在 lecture slides 的范围内，那么最自然的做法不是 log-mel、MFCC 或神经网络，而是直接使用 chap4 和 chap7 里的 Fourier transform / DFT / FFT 工具来描述局部频谱。语音虽然不是周期信号，但只要在整词上取一个短窗口，就可以把这段局部波形近似看成短时片段，并用 FFT 观察它的频率成分。

设第 $i$ 个样本的波形为 $x_i[n]$。取帧长 $L=20$ ms、帧移 $H=10$ ms、FFT 点数 $N=512$。第 $m$ 帧的离散频谱定义为
\[
X_{i,m}[k] = \sum_{n=0}^{L-1} x_i[n+mH] e^{-j 2\pi kn/N}, \qquad k=0,\dots,N-1.
\]
由于示例音频前部常带静音，课堂版检测器不只看词首一个窗口，而是在整词上滑动一个短时块。设每个块包含连续 $M$ 帧，则对应的局部幅度谱描述为
\[
P_i^{(r)}[k]
=
\frac{1}{M}\sum_{m=0}^{M-1}
\frac{|X_{i,r+m}[k]|}{\max_{\kappa}|X_{i,r+m}[\kappa]|},
\]
其中 $r$ 是块的起始帧。归一化的目的只是减弱录音音量差异，让判别主要依赖谱形而不是整体幅度。

对每个标签 $c\in\{g,b,d,z\}$，把训练集或参考样本中最有能量的短时块幅度谱取平均，得到该辅音的模板
\[
T_c[k] = \frac{1}{N_c}\sum_{i:y_i=c} P_i^{(\star)}[k].
\]
检测时，利用 chap4 中的互相关思想，把测试样本的局部谱和模板做归一化相关：
\[
\rho_{i,c}^{(r)}
=
\frac{\sum_k P_i^{(r)}[k]\,T_c[k]}
{\sqrt{\sum_k |P_i^{(r)}[k]|^2}\sqrt{\sum_k |T_c[k]|^2}}.
\]
再在所有滑动窗口上取最大值
\[
s_{i,c} = \max_r \rho_{i,c}^{(r)}.
\]
如果是单标签四分类任务，就取 $\hat c_i=\arg\max_c s_{i,c}$；如果保留一对多接口，则仍可在 dev 集上为每个标签选阈值 $\tau_c$。

这个版本只依赖以下课堂工具：
\begin{itemize}[nosep]
    \item 非周期信号的 Fourier transform 观点与局部频谱描述；
    \item DFT / FFT 的 MATLAB 实现；
    \item 时间平移下的滑窗搜索；
    \item 互相关 / 归一化相关作为模板匹配分数。
\end{itemize}
因此，严格课内版不再引入 log-mel、MFCC、负模板差分或额外学习到的特征组合。

\subsection{算法 1：课堂版 MATLAB 频谱模板检测}

\begin{verbatim}
Algorithm 1 短时幅度谱模板检测
Input: template waveforms, query waveform
Output: detected label

for each label c do
    scan the whole waveform with 20 ms frames and 10 ms hop
    pick the 4-frame block with the largest energy
    average the FFT magnitudes in this block to build template T_c
end for

for each query waveform do
    scan all candidate 4-frame blocks
    for each label c do
        compute normalized correlation between the block spectrum and T_c
        keep the maximum score s_ic over all blocks
    end for
    predict argmax_c s_ic
end for
\end{verbatim}

对应的 MATLAB 实现放在 \path{src/matlab_course_template_detector.m} 和 \path{src/matlab_course_spectrum_demo.m} 中。核心三步就是：\texttt{audioread} 读入语音，\texttt{fft} 求局部幅度谱，最后用归一化内积实现相关打分。

\begin{verbatim}
X = fft(frame, 512);
mag = abs(X(1:257));
score = (mag' * template) / (norm(mag) * norm(template));
\end{verbatim}

\subsection{课外扩展：局部窗对比模板}

当前项目里保留的 \texttt{local\_patch\_v1} 不是严格课内版本，而是一个为了提高分数而做的工程扩展：它会先定位 onset 邻域，再在局部窗上做更复杂的谱图搜索，并引入额外的统计描述和调参。因此，后文凡是提到 \texttt{local\_patch\_v1} 或其结果，都应理解为“课外扩展版模板法”，而不是上一节那个严格按课堂工具实现的 MATLAB 版本。

\subsection{课外对照 1：统计特征 MLP 基线}

MLP 不是主角，只是一个强一点的统计基线。它的输入不是原始波形，而是手工汇总后的 62 维统计特征：
\begin{itemize}[nosep]
    \item 13 维 MFCC 的均值和标准差；
    \item MFCC 一阶差分的均值和标准差；
    \item 10 个 coarse band-energy 特征。
\end{itemize}

模型结构是 \texttt{StandardScaler -> MLPClassifier(128, 64)}，用 early stopping 训练。和模板法相比，它不再依赖显式模板对齐，而是直接学习这些统计特征之间的组合关系。因为当前数据是单标签四分类，但阈值仍按 label 单独选，所以 MLP 的 exact-match 往往会比 label-wise accuracy 更能反映它是否“少乱报”。

\subsection{课外对照 2：CNN / AI 基线}

CNN 只作为更强的对照上界。输入是整词 log-mel spectrogram，网络是 3 个卷积块加自适应池化，再接一层小型全连接头。训练时使用带 \texttt{pos\_weight} 的 \texttt{BCEWithLogitsLoss}，并按 dev macro-F1 早停回退到最佳 epoch。

最终采用的调参配置是：
\[
\texttt{epochs}=10,\quad
\texttt{batch}=192,\quad
\texttt{lr}=10^{-3},\quad
\texttt{weight\_decay}=10^{-4},\quad
\texttt{patience}=5.
\]
它的作用只是提供一个“模型能学到多少局部时频模式”的参照，不影响本文的主要论证。

\section{讨论与局限}

这次重构最重要的收益，不是某个分数上升了，而是任务边界终于变清楚了。旧版更像“词里有没有出现某个浊辅音”，新版则更接近“一个整词里能不能检出目标浊辅音”，而严格课内版与课外扩展版也终于被分开陈述：前者负责完成题目要求，后者负责展示继续工程化以后能提升到什么程度。

这套方案仍然有三个局限：
\begin{itemize}[nosep]
    \item 语音本身仍是整词级音频，不是人工切好的 phone segment；严格课内版只能依赖滑窗搜索，扩展版则仍需要 onset 近似对齐。
    \item /z/ 的词数仍然是数据集构造的瓶颈，想继续扩规模就得放宽单词池或换更大的语料。
    \item /g/ 和 /d/ 的 burst 很短，简单 FFT 模板容易和邻近类别混淆；扩展版虽然更强，但也仍会受对齐误差影响。
\end{itemize}

\section{结论}

本文完成了 Topic 2 的一次重新组织，核心结论如下：
\begin{itemize}[nosep]
    \item 通过“首字母与首个非元音音素严格对应”的规则，新数据集把任务收紧到了首浊辅音检测；
    \item 严格课内版可以只用短时 FFT 幅度谱和相关检测，在 MATLAB 中完成四类浊辅音的频谱描述与示例检测；
    \item 课外扩展版 \texttt{local\_patch\_v1} 在当前四类任务上取得最佳整体结果，macro-F1 为 0.546，exact-match 为 0.550；
    \item MLP 的 macro-F1 为 0.526，CNN 为 0.222，说明更强的深度模型并没有自动取代一个做对了局部频谱建模的模板法；
    \item /z/ 最容易、/g/ 最难，这和声学图里的持续摩擦与短促爆破完全一致；
    \item 这份报告的重点已经从“跑一个神经网络”转成“解释为什么这些辅音会这样表现”。
\end{itemize}
