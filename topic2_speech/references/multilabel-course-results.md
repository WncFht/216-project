## Why this upgrade exists

The previous version classified only one initial consonant per word.  This was
too narrow and could overfit word identity.  This upgrade changes the task to
multi-label voiced-consonant presence detection: a word can contain several
voiced consonants, e.g. `backward -> /b,w,r,d/`, `marvin -> /m,r,v,n/`.

## Covered voiced consonants

Covered by selected real Speech Commands words:

`/b/`, `/d/`, `/g/`, `/j/`, `/l/`, `/m/`, `/n/`, `/r/`, `/v/`, `/w/`, `/z/`, `/zh/`.

Not reliably covered by this dataset: `/ng/`, `/dh/`, `/dzh/` as separate phone-level labels.  These require TIMIT or a custom recording set.

## Interpretation

The course baseline is intentionally simple and explainable: it uses the same
logic as a spectrum analyzer and a band-energy template detector.  It performs
worse than CNN because word-level multi-label detection contains variable phone
positions and several consonants per word.  The key conclusion is not that the
course method is useless; it gives an interpretable baseline and makes the
spectrum visible, while CNN is needed to learn richer time-frequency patterns.
